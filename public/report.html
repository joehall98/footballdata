Data Logging Tool - Report


Ideas, Intentions and Aims

I knew I wanted to get data externally (via API) for this project so I decided to look on Any-API (Reference 1) where I came across the Football-Data API (Reference 2). It has a whole load of great data on their free package that includes 12 different competitions and includes stats such as fixtures results and league tables. From this information I knew I wanted to display the English leagues data as I have a passion for Football. I instantly began to think about how I could display the data from the API along with some financial data that I collected myself from Spotrac (Reference 3) and The Guardian (Reference 4). I decided to find all 20 Premier League teams wage bill and profit/loss accounts from the last financial year. I thought this would be great data to compare with how well each individual team is preforming in the league trying to answer the age-old question in football; does money equal success?


I wanted to make a 2 horizontally stacked bar chart (Image 1) one bar showing the teams points and the other showing how many games they had played. The idea was to have the points bar either green/ red depending on whether the team made a profit, or a loss and the team played bar to be greyscale depending on how much they spend on wages from black being the most to white being the least (Image 2). However, I ran out of time and was unable to complete the played bars to visualise this, however I added in a votes line on my chart so my users can log data from the webpage (Image 3). There is a dropdown menu above my chart that contains all 20 teams from the league, users can use this to select their favourite team then refresh the page and what their vote being added. I thought this was good as users now have more interactivity with my design and they are now another data source.


Data Sources

I had a variety of different data sources for my project. The first being the Football-Data API I used. I have included a link to the API on the sidebar of the site housing the data logging tool as it is a really great API with loads of great data, and I had fun using it. Secondly, I gathered Financial information for all 20 of the teams and created a database within my files and passed my information into it by running arrays of variables through for loops and storing them in my database as JavaScript objects. I gathered information from Spotrac, The Guardian and a site called Financial Football News (Reference 5). The vote is my last data source that comes from users. I think all 3 of these data sources are vastly different so gives my chart some uniqueness to it in my opinion as nobody else will have all this data in one place.
